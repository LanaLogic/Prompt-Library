**Основные параметры оптимизации:**

- **Task** (задача): какое действие необходимо делать?

- **Context** (контекст): условия, аудитория и ограничения;

- **Input** (ввод): какие данные обрабатываются? Какой ответ следует считать корректным?

- **Output** (вывод): формат результата.

**Составляющие метода:**

- **Concise** (краткость): цель фиксируется в 1-2 предложениях, убирая размытость.

- **Logical** (логичность): задаётся повторяемый порядок шагов, что уменьшает фактор случайности.

- **Explicit** (явность): указывается точный формат выхода для интеграции в рабочий процесс, сопровождаемый схемой и примерами.

- **Adaptive** (адаптивный): один шаблон работает для разных режимов (драфт/продакшн/деплой) с разным требованием по качеству.

- **Reflective** (рефлексивный): встроенный контроль качества и самокоррекция.
  
  Разделяй:

- **tags** → тема
- **use_case** → задача
- **type** → тип файла

## Переменными могут быть

- Данные: требования, описания продукта, отрывки документов;

- Роль, целевая аудитория, интонация;

- Формат и ограничения: количество пунктов, длина текста, язык, тип артефакта.
